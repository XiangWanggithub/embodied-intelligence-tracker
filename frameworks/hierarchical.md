# 分层架构

> 多层次模型协作，每层负责不同抽象级别的任务

## 典型分层

```
任务层 (Task)     ← LLM / VLM：任务分解
      ↓
技能层 (Skill)   ← 技能库：Pick, Place, Push...
      ↓
动作层 (Action)   ← 的低层策略：关节控制、位姿跟踪
```

## 代表性工作

- **Octo**: 可插拔的技能模块
- **pi_0.5**: 高层推理 + 低层控制
- **RT-Frame**: 框架层面的分层设计

## 优缺点

### 优点
- 可复用：技能层可在多个任务间共享
- 可解释：每层职责清晰
- 易调试：分层定位问题

### 缺点
- 接口可能成为瓶颈
- 端到端效果可能更好
- 需要设计层间通信
