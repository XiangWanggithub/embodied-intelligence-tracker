# System 1 / System 2 架构

> 快系统（实时控制）+ 慢系统（推理规划）

## 核心概念

- **System 1**: 快速反射式控制，延迟 < 10ms
  - 例子：低层策略模型、阻抗控制
- **System 2**: 慢速推理规划，延迟秒级
  - 例子：LLM 任务分解、视觉语言模型推理

## 典型实践

| 模型 | System 1 | System 2 |
|------|----------|----------|
| pi_0.5 | 低层扩散策略 | 高层 VLA 推理 |
| HELIX | 全身控制策略 | VLM 规划 |
| 人在回路 | 实时执行 | 监督纠正 |

## 优势

1. **效率**：简单任务用 System 1 直接搞定
2. **泛化**：复杂任务交给 System 2 推理
3. **安全**：可加入安全仲裁层

## 挑战

1. 接口设计：两系统通信协议
2. 同步机制：何时切换？
3. 端到端 vs 分层的权衡
